# Skill: screenplay-page

사용자가 제공한 줄거리·시놉시스·대본(또는 그 일부)을 받아 **완성된 영화 시나리오 패키지**를 단일 HTML 페이지로 출력하는 스킬.

---

## 트리거

`/screenplay-page` 또는 사용자가 줄거리·시놉시스·대본을 붙여넣고 시나리오 페이지 제작을 요청할 때.

---

## 실행 순서

아래 단계를 **순서대로, 빠짐없이** 수행한다. 각 단계 결과는 내부적으로 누적하여 최종 HTML에 반영한다.

---

### STEP 0 — 입력 수집 및 분석

1. 사용자가 제공한 텍스트(줄거리/시놉시스/대본/아이디어 메모 등 형태 무관)를 그대로 수용한다.
2. 첨부 이미지가 있으면 이미지를 시각적으로 분석하여 **지배 색상 5개, 보조 색상 3개, 강조 색상 2개**를 HEX 값으로 추출한다. 이미지가 없으면 스토리 무드에서 팔레트를 창작한다.
3. 입력 텍스트의 언어를 감지한다. 출력 HTML 텍스트는 **입력 언어와 동일한 언어**로 작성한다(기본: 한국어).

---

### STEP 1 — 스토리 완성 (리라이팅)

> 노벨문학상 수준의 문학적 깊이 + 넷플릭스 오리지널 파일럿 작가의 상업적 긴장감을 결합해 리라이팅한다.

- 플롯 홀·논리적 공백을 메운다.
- 캐릭터 동기와 내면 갈등을 구체화한다.
- 감정 곡선을 강화하고, 반전·페이소스·카타르시스를 설계한다.
- 주제적 울림이 있는 오프닝 이미지와 클로징 이미지를 확정한다.
- 결과: **완성된 스토리 내러티브** (산문체, 2,000자 이상).

---

### STEP 2 — 구조화 섹션 생성

다음 8개 섹션을 순서대로 작성한다.

#### 2-A. 로그라인
- 주인공 + 목표 + 장애물 + 세계관을 25~40자 이내 한 문장으로 압축.
- 감정 훅과 아이러니가 느껴져야 한다.

#### 2-B. 시놉시스
- 3단락 구성: 발단(세계와 인물 소개) → 갈등(핵심 사건·전환점) → 결말(해소 또는 열린 결말).
- 각 단락 150~250자.

#### 2-C. 인물관계도
- 주요 인물 목록과 관계를 ASCII 다이어그램 또는 설명형 텍스트로 표현.
- HTML 렌더링 시 `<pre>` 블록 또는 CSS Flexbox 카드 배치로 시각화.

#### 2-D. 캐릭터 시트
각 주요 인물에 대해:
- 이름 / 나이 / 역할 (주인공·조력자·반동인물 등)
- 욕망 (Want) vs. 필요 (Need)
- 결함 (Flaw) 과 강점 (Strength)
- 핵심 대사 1줄

#### 2-E. 막 구조 (3막 또는 5막)
- 1막: 설정 → 촉발 사건 → 1막 전환점
- 2막: 상승 갈등 → 중간점 → 어둠의 밤
- 3막: 클라이맥스 → 해소 → 여운
- 각 막의 페이지 범위(예: 1막 1~25p)와 핵심 사건 1~3개.

#### 2-F. 비트 트리트먼트
- 전체 스토리를 15~25개 비트(장면 단위 사건)로 나열.
- 각 비트: `[비트 번호] 장소 / 시간 — 사건 요약 (감정 변화 방향 ↑/↓)`

#### 2-G. 상징 & 모티프
- 반복되는 이미지, 색, 소품, 음악, 대사 패턴 5~8개.
- 각 상징이 주제와 어떻게 연결되는지 1~2줄 해설.

#### 2-H. 영화적 무드 보드 (텍스트 버전)
- 장르, 톤, 촬영 스타일, 색채 팔레트 설명.
- 참조 영화 3~5편과 구체적 유사 장면/연출 설명.

---

### STEP 3 — 영화적 인사이트

6개 축으로 분석한다.

| 축 | 내용 |
|---|---|
| **주제** | 핵심 질문, 보편적 진실, 철학적 함의 |
| **구조** | 비전통적 구조 사용 여부, 서사 혁신 포인트 |
| **시각 문법** | 카메라 무브먼트, 편집 리듬, 렌즈 선택 제안 |
| **사운드 & 음악** | 다이제틱/넌다이제틱 사운드 활용, 스코어 무드 |
| **연기 디렉션** | 각 주요 씬의 배우 연기 키워드 (내면화/외현화 균형) |
| **레퍼런스** | 영화·문학·미술·음악 레퍼런스 및 차별화 전략 |

---

### STEP 5 — HTML 페이지 생성

위 모든 내용을 **단일 `.html` 파일** 하나로 조립한다.

#### 레이아웃 구조

```
┌─────────────────────────────────────────┐
│  HERO  (제목 + 로그라인 + 무드 이미지 배경)   │
├──────────┬──────────────────────────────┤
│ SIDEBAR  │  MAIN CONTENT               │
│ (챕터    │  (스크롤 섹션들)              │
│  버튼    │                              │
│  고정)   │                              │
└──────────┴──────────────────────────────┘
│  FOOTER  (컬러칩 팔레트 + 크레딧)           │
└─────────────────────────────────────────┘
```

#### 필수 기능

1. **챕터 사이드바** — 모든 섹션 링크 버튼, `position: sticky` 또는 `position: fixed`, 현재 뷰 섹션 하이라이트 (IntersectionObserver 사용).
2. **전체 스크롤** — 섹션 간 부드러운 스크롤 (`scroll-behavior: smooth`).
3. **컬러칩 팔레트** — 푸터에 추출된 색상 칩 시각 표시 (HEX 값 레이블 포함).
4. **반응형** — 모바일에서 사이드바는 상단 고정 탭 or 햄버거 메뉴로 전환.
5. **인쇄 모드** — `@media print` 시 사이드바 숨김, 컨텐츠 1열 출력.
6. **타이포그래피** — Google Fonts 로드 (`Playfair Display`, `Inter`, 한국어면 `Noto Serif KR`/`Noto Sans KR` 추가).

#### 섹션 ID 및 순서

```
#hero          → 히어로 (제목 + 로그라인)
#story         → 완성된 스토리
#logline       → 로그라인
#synopsis      → 시놉시스
#characters    → 인물관계도 + 캐릭터 시트
#act-structure → 막 구조
#beat-sheet    → 비트 트리트먼트
#symbols       → 상징 & 모티프
#mood          → 영화적 무드 보드
#insights      → 영화적 인사이트
#palette       → 컬러 팔레트 (푸터)
```

#### 코드 품질 기준

- 외부 라이브러리는 Google Fonts CDN, 순수 Vanilla JS만 허용 (React/Vue 금지).
- 인라인 `<style>` + `<script>` 단일 파일로 완결.
- 시맨틱 HTML5 태그 사용 (`<section>`, `<article>`, `<aside>`, `<nav>` 등).
- 다크 배경 + 영화적 타이포그래피 기본 테마 (추출 색상으로 커스터마이즈).
- 전체 HTML이 **완전히 닫힌 유효한 파일**이어야 한다. 중간에 잘리면 안 된다.

---

## 출력 형식

1. 완성된 HTML 코드를 코드블록 ` ```html ... ``` ` 으로 출력한다.
2. HTML 코드 **앞에** 다음 요약을 3~5줄로 먼저 출력한다:
   - 제목
   - 로그라인
   - 추출된 주요 색상 (HEX 3개)
   - 총 비트 수 / 막 구조 형식
3. 사용자가 원하면 HTML을 파일로 저장하거나 추가 수정을 제안한다.

---

## 주의사항

- 입력이 짧거나 미완성이어도 창작적으로 완성한다. "내용 부족"을 이유로 거절하지 않는다.
- 폭력·혐오·불법 콘텐츠 요청은 거절하되, 성인 드라마·스릴러·공포 장르는 창작적 맥락에서 수용한다.
- 저작권 있는 원작을 **그대로 복제**하지 않는다. 영감은 받되 오리지널 창작을 원칙으로 한다.
- HTML 출력이 너무 길어질 경우 Write 도구를 사용해 파일로 저장하고 경로를 알려준다.
